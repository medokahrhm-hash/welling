# دليل تشغيل Nexus Core على الهاتف (أندرويد)

## لماذا لا يمكن تشغيل ملف .py مباشرة على الهاتف؟
الهاتف لا يملك مفسّر بايثون مدمج بالنظام، ولا يفهم تطبيقات سطح المكتب.
لازم نحوّل المشروع إلى ملف **APK** قابل للتثبيت، وهذا ما تفعله أداة
**Buildozer** (تبني بيئة أندرويد كاملة حول كود بايثون تلقائياً).

## الطريقة الموصى بها: البناء عبر كمبيوتر (مرة واحدة فقط)

### المتطلبات (على كمبيوتر لينكس، أو WSL على ويندوز)
```bash
sudo apt update
sudo apt install -y python3-pip build-essential git python3-dev \
    ffmpeg libsdl2-dev libsdl2-image-dev libsdl2-mixer-dev libsdl2-ttf-dev \
    libportmidi-dev libswscale-dev libavformat-dev libavcodec-dev zlib1g-dev openjdk-17-jdk

pip3 install --user buildozer cython
```

### خطوات البناء
1. ضع الملفات الأربعة (`main.py`, `database.py`, `crypto_utils.py`,
   `wheel_widget.py`) و`buildozer.spec` في نفس المجلد.
2. من داخل المجلد:
```bash
buildozer android debug
```
3. أول مرة ستنزّل Buildozer تلقائياً Android SDK وNDK (يأخذ وقتاً طويلاً،
   من 20 دقيقة إلى ساعة حسب سرعة الإنترنت).
4. عند الانتهاء، ستجد الملف هنا:
```
bin/nexuscore-1.0-arm64-v8a-debug.apk
```
5. انسخ هذا الملف لهاتفك (عبر كابل USB، أو رفعه على Google Drive ثم تنزيله)
   وثبّته (فعّل "السماح بالتثبيت من مصادر غير معروفة" في إعدادات أندرويد أولاً).

### بديل أسرع: بدون تثبيت أي شيء على جهازك
يمكنك استخدام خدمة **GitHub Actions** لبناء الـ APK سحابياً مجاناً:
1. ارفع المجلد كمستودع GitHub.
2. أضف workflow جاهز (`kivy/buildozer-action`) يبني الـ APK تلقائياً عند كل push.
3. تنزّل ملف الـ APK الناتج من تبويب "Actions" في المستودع، مباشرة من هاتفك.

هذا الخيار مناسب جداً لو لا تملك كمبيوتر لينكس قوي، لأن البناء يتم بالكامل
على سيرفرات GitHub.

## ما الذي تغيّر فعلياً عن نسخة PyQt5 ولماذا

| الجزء | PyQt5 (سطح مكتب) | Kivy (هاتف) | السبب |
|---|---|---|---|
| الرسم | `QPainter` + `QPainterPath` | `canvas` + `Mesh` (GPU مباشر) | PyQt5 لا يُبنى أصلاً على أندرويد |
| الإدخال | `keyPressEvent` (أسهم لوحة المفاتيح) | `on_touch_down/move/up` (سحب ونقر) | لا توجد لوحة مفاتيح فعلية على الهاتف |
| التشفير | `cryptography` (Fernet) | `pycryptodome` (AES-256-CTR) | بناء `cryptography` لأندرويد غير مستقر |
| اختيار الملفات | `QFileDialog` | `plyer.filechooser` | يفتح منتقي ملفات أندرويد الأصلي (SAF) |
| قاعدة البيانات | ملف بجانب الكود | `App.user_data_dir` | على أندرويد كل تطبيق معزول بمجلد بيانات خاص به |
| فتح برامج خارجية (.exe) | `subprocess.Popen` | **غير مدعوم** | أندرويد لا يشغّل ملفات .exe إطلاقاً (نظام مختلف تماماً) |

## ملاحظة مهمة عن ميزة "ربط وتشغيل برامج .exe"
هذه الميزة من النسخة الأصلية **لا معنى لها على أندرويد** لأن ملفات `.exe`
هي برامج ويندوز فقط. أبقيت الكود قابلاً لربط أي ملف عام (`add_real_file`)،
لكن "تشغيله" كبرنامج تنفيذي محذوف لأنه غير ممكن تقنياً على الهاتف. لو
الهدف فتح ملفات عادية (صور، PDF...) بالتطبيق المناسب، هذا ممكن عبر
`plyer` أو نوايا أندرويد (Intents)، أخبرني إذا أردت أضيفها.

## التحكم بالتطبيق على الهاتف
- **سحب يمين/يسار** على القرص = تدوير العجلة (بديل أسهم لوحة المفاتيح)
- **نقرة بسيطة** على القرص = فتح/تنفيذ العنصر النشط (بديل Enter)
- **زر ⬅ رجوع** = الخروج لمجلد أعلى (بديل Backspace)
- **زر 🔒 قفل/تشفير** = تشفير الملف النشط حالياً
- **زر ➕ إضافة ملف** = فتح منتقي ملفات أندرويد وربط ملف حقيقي
